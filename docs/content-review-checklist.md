# Content Review Checklist

Use this checklist when reviewing blog post PRs.

## Technical Accuracy

- [ ] Commands and examples are correct
- [ ] AWS services are used appropriately
- [ ] No misleading or risky advice

## Clarity and Structure

- [ ] Clear introduction that explains the problem
- [ ] Logical flow of sections
- [ ] Simple language, no unnecessary jargon
- [ ] Conclusion or summary included

## Formatting

- [ ] Uses correct frontmatter fields
- [ ] Headings are properly structured
- [ ] Code blocks are formatted correctly
- [ ] Links work and point to correct resources

## Audience Fit

- [ ] Suitable for beginners or clearly states required knowledge
- [ ] Explains why the topic matters in real use cases

## Final Check

- [ ] No unrelated file changes
- [ ] PR description explains what was added
